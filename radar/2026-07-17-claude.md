## Highest-impact

### 1. Auto mode could override a PreToolUse hook's `ask` — now floored. Harness has NO exposure (it denies via exit-2), which the fix validates
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.211, July 15, 2026 · seen 2026-07-17
- **What it is**: *"Fixed auto mode overriding a PreToolUse hook's `ask` decision for unsandboxed Bash
  — a hook `ask` now floors the decision at a prompt."* Previously, under auto mode, a PreToolUse hook
  that returned `ask` could be overridden and the tool allowed to run without the prompt.
- **Why it matters here**: This is the exact enforcement layer every kit gate lives on. I grepped all
  three kits' `hooks/`: the gates (`gate_pipeline`, `gate_git`, `gate_test_coverage`,
  `guard_agent_spawn`, `guard_harness_selfmod`, `gate_packaging_decision`, …) block with **`sys.exit(2)`
  hard-deny**, and the only `hookSpecificOutput` emitters are the `session_status.py` status hooks —
  **no kit gate returns `ask`**. The now-fixed bug applied specifically to `ask` decisions, so the
  harness was **never exposed**: an exit-2 deny is a hard block that auto mode could not override. The
  fix *validates* the design choice — record so nobody "hardens" by converting a gate to `ask`.
- **Recommendation**: **ignore (no exposure; confirms exit-2-deny design)** · note the CC floor →
  **≥ 2.1.211** for anyone who *does* rely on `ask` semantics. · **Status**: NEW

### 2. Per-session cap on subagent spawns (default 200) — free defense-in-depth for `guard_agent_spawn` / anti-re-delegation
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.212, July 17, 2026 · seen 2026-07-17
- **What it is**: *"Added a per-session cap on subagent spawns (default 200, override with
  `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`) to stop runaway delegation loops; `/clear` resets the
  budget."*
- **Why it matters here**: The harness prevents runaway/recursive delegation at the **prompt+hook**
  layer (`guard_agent_spawn` + V14b `deny: Agent(project-manager)` + the anti-re-delegation gate). The
  platform now adds a **runtime hard ceiling** on total spawns per session — the same failure mode
  (a specialist fan-out loop) is now also capped by the runtime. No kit change required; optionally the
  kits could pin a *lower* `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION` than 200 for the PM/specialist flow,
  but that's a tuning decision for you, not a correctness gap.
- **Recommendation**: **ignore (confirms design)** · optional: note the env var in the kit's
  orchestration docs as a belt-and-suspenders knob (~15 min if adopted). · **Status**: NEW

### 3. Approval-message integrity: permission previews now neutralize bidi/zero-width/look-alike chars
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.211, July 15, 2026 · seen 2026-07-17
- **What it is**: *"Fixed permission previews relayed to chat channels not neutralizing
  bidirectional-override, zero-width, and look-alike quote characters, so tool inputs cannot visually
  alter the approval message."*
- **Why it matters here**: Lands on the **two-level user-gate / no-fabricated-approval** invariant
  (same family as last week's item 6). A specialist can't smuggle a homoglyph/bidi payload into a tool
  input to make the human approve something other than what runs. The harness enforces the *semantics*
  of that gate in prompt; the platform now closes the *rendering* side. Defense-in-depth, no kit change.
- **Recommendation**: **ignore (confirms design)** · same **≥ 2.1.211** floor note. · **Status**: NEW
