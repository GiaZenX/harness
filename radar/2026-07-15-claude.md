## Highest-impact

### 1. Mythos 5 is now GA — but it is NOT a higher tier; do not re-pin the §11 escalation model
- **Source**: model-deprecations table (`claude-mythos-preview` retires **July 21, 2026** → migrate to
  `claude-mythos-5`) https://platform.claude.com/docs/en/about-claude/model-deprecations · positioning
  https://www.anthropic.com/news/claude-fable-5-mythos-5 + https://platform.claude.com/docs/en/about-claude/models/introducing-claude-fable-5-and-claude-mythos-5 · seen 2026-07-15
- **What it is**: Claude Mythos 5 (`claude-mythos-5`) reaches GA as the Mythos-Preview successor.
  Verified against the official launch note (press-lead → official-confirmed, per the standing rule):
  Mythos 5 is the **same capability ceiling as Fable 5**, a *limited-release cybersecurity variant*
  (Project Glasswing) with Fable's safety classifiers removed — **not** a more-capable model above Fable.
- **Why it matters here**: `team-kits/model_tiers.yaml` pins `fable` as the Claude-side "Mythos-class,
  above opus" §11 escalation model. A new `mythos-5` GA is exactly the standing-watch trigger ("next
  Mythos-class GA"), and a naive read of the name would suggest bumping the escalation pin. It should
  **not** be bumped: same ceiling, gated/limited access, cyber-only positioning. Recording this so a
  future triage (or the codex-watcher mirror) doesn't re-tier `fable → mythos-5`.
- **Recommendation**: **ignore (no re-tier)** — `fable` stays the escalation pin. Record in `decided.md`.
- **Status**: NEW

- **Source**: https://code.claude.com/docs/en/changelog — 2.1.206, July 9, 2026 · seen 2026-07-15
- **What it is**: A new `/doctor` check "proposes trimming checked-in `CLAUDE.md` files by cutting
  content Claude could derive from the codebase."
- **Why it matters here**: The harness ships a **deliberately-authored `CLAUDE.md` shim** (the Codex/
  neutral-source contract; HARNESS_LOG 2026-07-14). `/doctor` may flag that shim's content as
  "derivable" and propose cutting it — a false positive against a file whose contents are load-bearing
  for the source-format bridge. No auto-edit (it only *proposes*), so the risk is a human accepting a
  bad suggestion, not silent drift.
- **Recommendation**: **watch** · no effort now — note in the kit's CLAUDE.md/HARNESS_LOG that the shim
  is intentional and should survive a `/doctor` trim proposal.
- **Status**: NEW

## Confirmations — platform now backstops harness invariants (no change; ensure CC ≥ 2.1.210)

### 6. Fabricated-approval / prompt-injection / re-delegation hardening
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.205 (Jul 8), 2.1.203 (Jul 7), 2.1.210
  (Jul 14) · seen 2026-07-15
- **What it is**: (a) 2.1.205 — "Background task notifications now explicitly state that no human input
  has occurred, preventing fabricated in-transcript approvals from being acted on"; (b) 2.1.210 —
  "Hardened the Agent tool against indirect prompt injection via content a subagent read"; (c) 2.1.203 —
  "agents are now less likely to re-delegate their entire task to another subagent."
- **Why it matters here**: These land precisely on harness invariants that are today *prompt-enforced* —
  the two-level user-gate (a specialist can't manufacture an approval), and the anti-re-delegation gate
  (`guard_agent_spawn` + V14b `deny: Agent(project-manager)`). The platform now enforces the same at the
  runtime layer: defense-in-depth for free. No kit change; bump the floor to **Claude Code ≥ 2.1.210**.
- **Recommendation**: **ignore (confirms design)** · note the version floor. · **Status**: NEW

### 7. Worktree-isolated subagents no longer leak git-writes to the main checkout
- **Source**: https://code.claude.com/docs/en/changelog — 2.1.203 (Jul 7) + 2.1.210 (Jul 14) · seen 2026-07-15

- **Recommendation**: **ignore (not used today)**. · **Status**: NEW
